## 1. これは何か（全体像）

大阪大学軽音部の**部室予約Webサイト**です。2026年に旧サイトからリニューアルしました。

- **本番サイト**: https://oumkeion-reservation-app.web.app
- **旧サイト**（使用停止）: https://oum-keion-bushitsu.web.app
- **予約はログイン不要**。誰でもバンド名・記入者名を入れて予約できます。
- **Googleログインは管理者専用**（固定枠・音出し禁止・部のイベントの作成、確認なし削除など）。

### 主要サービス（すべて無料枠で運用）
| 役割 | サービス | 費用 |
|---|---|---|
| サイト表示 | Firebase Hosting | 無料 |
| データベース | Firebase Firestore | 無料枠 |
| 管理者ログイン | Firebase Authentication (Google) | 無料 |
| 講義棟データ自動取得 | GitHub Actions（毎晩） | 無料（公開リポジトリ） |
| ソースコード管理 | GitHub | 無料 |

**お金は一切かかりません。** 課金される設定にはなっていません（後述の「支出上限$0」を維持すれば構造的に課金不可）。

## 3. 日常の管理タスク

### 3-1. 管理者としてサイトを使う
1. サイトを開く → 右上「管理者ログイン」→ 管理者のGoogleアカウントでログイン。
2. ログインすると、固定枠・音出し禁止・部のイベントの作成／削除ができます。
3. 「固定枠を一括登録」ボタンで、毎週の固定枠をまとめて登録できます。

### 3-2. 固定枠の申請を承認する
- 「固定枠申請」タブに未処理の申請が出ます。
- 承認すると実際の固定枠が変更・削除されます。拒否も可能。
- 申請状況は一般部員も閲覧できますが、承認・拒否は管理者だけ。

### 3-3. 予約データのメンテナンス
- 60日より前の古い予約は**自動で削除**されます（誰かがサイトを開いたとき）。手動対応は基本不要。
- 希望枠の自動確定も同様に自動で走ります。

## 4. 困ったときの対処

| 症状 | 原因と対処 |
|---|---|
| **「講義棟データが36時間以上更新されていません」警告** | 夜間スクレイパーが失敗している。→ 6章参照。まずGitHub Actionsのログを確認。大学のID/PW変更、SAキー失効が主な原因。 |
| **サイトが真っ白／エラー** | ブラウザのキャッシュをクリア。直らなければデプロイ直後の不具合の可能性 → 7章でロールバック。 |
| **管理者ログインできない** | そのGoogleアカウントが `admins` コレクションに登録されているか確認（2-2）。 |
| **予約が保存できない（重複エラー）** | 同じ時間・部屋に既存予約がある。仕様。 |
| **スマホで予約画面が開かない** | 長押ししてからなぞる操作。ユーザーへの周知不足の可能性。 |
| **課金が心配** | GitHub/Firebaseとも支払い方法未登録なら課金されない。Firebase Console → 使用量とお支払い で確認。 |

- **カレンダーは常に今日が左端**の7日間表示。
- **LINE bot・メール通知は未設定**（10章）。

## 6. 講義棟スクレイパー（夜間自動取得）

大学の施設予約システムから講義棟の使用状況を取得し、「音出し禁止帯」を計算する仕組みです。

- 実装: `public/scraper_requests.py`
- 実行: GitHub Actions `nightly-scrape.yml`
- **スケジュール: 毎日 UTC 15:30（＝日本時間 深夜0:30）** に自動実行。手動実行も可能。
- 処理: 大学システムにログイン → 60日分×10部屋を取得 → `settings/lectureHall`（Firestore）に書き込み＋`public/htmls/lecture-hall.json`にバックアップコミット。

### スクレイパーが止まったときの復旧手順
1. GitHub → `oumkeion/reservation-app` → **Actions**タブ → 「Nightly Scrape」のログを確認。
2. よくある失敗原因:
   - **大学のID/PWが変わった** → `.env` とGitHub Secretsの `RESV_USER_ID` / `RESV_PASSWORD` を更新。
   - **SAキーが失効した** → Firebaseで再生成し、Secretの `FIREBASE_SERVICE_ACCOUNT` を更新（2-4参照）。
   - **大学サイトのHTML構造が変わった** → `scraper_requests.py` のパース部分を修正。
3. 修正後、Actionsタブから「Run workflow」で手動実行して成功を確認。

> 💡 60日間コミットが無いとGitHubがスケジュールを自動停止しますが、このワークフローは毎晩JSONをコミットするので回避されています。


## 8. データ構造（Firestore）

| コレクション | 用途 |
|---|---|
| `events` | 全予約。`{title, start, end, extendedProps:{type, editor, comment, recurrenceGroup, source}}` |
| `slotLocks` | 二重予約防止の占有ロック。docID=`YYYY-MM-DD_HHMM`（JST15分）。**更新禁止＝排他制御の要** |
| `bands` | バンド一覧。`{name, category, songs, performanceDate, representative, status}` |
| `slotIntents` | 狙い表明（この枠を使いたいという宣言） |
| `logs` | 操作ログ。作成のみ可、更新・削除不可（改ざん防止） |
| `posts` | 部誌の投稿 |
| `admins` | 管理者メール（ドキュメントIDにメールアドレス） |
| `fixedSlotRequests` | 固定枠のキャンセル・変更申請 |
| `settings/lectureHall` | スクレイパーが書き込む講義棟データ（クライアントは読み取りのみ） |
| `settings/notice` | お知らせ（管理者のみ書き込み） |
| `dailyMemos` | 日毎のメモ（管理者のみ） |

### 予約種別（`eventTypes.js`）
`confirmed`(確定/ピンク), `request`(希望/薄緑), `fixed`(固定/薄黄, 管理者), `personal`(個人/水色), `no-sound`(音出し禁止/灰, 管理者・自動), `club-event`(部イベント/薄紫, 管理者)

### セキュリティルールの要点（`firestore.rules`）
- 通常枠（confirmed/request/personal）は**サーバー側でも**形式と時間制約を検証（クライアント改ざん対策）。
- 保護枠（fixed/no-sound/club-event）の作成・削除は管理者のみ。
- `slotLocks` は作成可・**更新不可**（二重予約防止）。60日超の古い予約は誰でも削除可（自動掃除用）。
- バンドの更新は「name/songs/performanceDate/category の編集」か「解散」のみ許可。

## 9. 旧サイトからの変更点まとめ

旧サイト（oum-keion-bushitsu）は別のFirebaseプロジェクト・別の作りでした。主な違い：

| 項目 | 旧サイト | 新サイト |
|---|---|---|
| 予約枠 | 「朝・昼・17-19」など固定の時間帯名 | 15分単位で自由にドラッグ選択 |
| カレンダー | 曜日固定・枠ベースのリスト | 今日を左端にした7日間タイムグリッド |
| 二重予約 | 防止機構が弱い | `slotLocks` で原子的に防止 |
| バンド管理 | なし | カテゴリ別一覧・本番日順・演奏曲登録 |
| 希望枠 | 手動確定 | 競合なしなら自動で確定枠へ格上げ |
| 固定枠申請 | なし | 一般部員が申請・全員が閲覧・管理者が承認 |
| 講義棟連携 | あり | あり（毎晩自動取得、音出し可否のパース改善） |
| スマホ操作 | — | 長押しドラッグに最適化 |
| 部誌・操作ログ | — | 追加 |
| ホスティング費用 | — | 完全無料（公開リポジトリ＋支出上限$0） |

### 旧データの移行について
2026-07-03に旧サイトの今後の予約（確定枠・固定枠）を新サイトへ移行済み。移行したデータには `extendedProps.source: "legacy-migration"` の印が付いています（万一やり直す場合の目印）。詳細は `plan/migration-report-2026-07-03.md` を参照。
