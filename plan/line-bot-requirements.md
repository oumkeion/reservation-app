# LINE bot（軽音部エージェント）要件定義

最終更新: 2026-06-12

## 1. 目的

各バンドのLINEグループに常駐し、部室予約まわりの事務作業を代替・効率化する。
**完全無料**（クレジットカード不要）で運用でき、代替わりで引き継げること。

## 2. 無料運用の制約と前提

| 項目 | 無料枠 | 影響 |
|---|---|---|
| LINE 応答メッセージ（reply） | **無制限・無料** | コマンド応答はいくら使ってもタダ |
| LINE プッシュ配信（push） | **月200通まで無料** | 前日リマインドが消費（予約のある日のみ配信して節約） |
| GAS（ホスティング） | 無料・カード不要 | URL Fetch 2万回/日、実行90分/日 — 部活規模では余裕 |

## 3. ホスティング選定（再吟味の結論: Google Apps Script）

| 観点 | **GAS（採用）** | Cloudflare Workers | Firebase Functions |
|---|---|---|---|
| 費用・カード | 無料・不要 | 無料・不要 | Blaze必須（カード必要） |
| 調エモン（Sheets）連携 | ◎ ネイティブ | △ API+共有設定が必要 | △ 同左 |
| Googleエコシステム統一 | ◎ | ✕ 別プラットフォーム | ◎ |
| 定期実行 | ◎ 時間主導トリガー | ◎ cron | ◎ |
| LINE署名検証 | **✕ 不可**（下記参照） | ○ | ○ |

日程調整（調エモンシートの複製・集計読取）が要件のため、Sheets/Drive にネイティブアクセスできる GAS が最適。
コードは `gas/` ディレクトリでリポジトリ管理し、`clasp` で GAS に同期する。

### セキュリティ上のトレードオフ（重要）
GAS の Web App は HTTPヘッダを読めないため、LINE の署名検証（x-line-signature）ができない。
**代替策**: Webhook URL に `?token=<長い乱数>` を付け、Script Properties の `WEBHOOK_TOKEN` と照合する。
残余リスク: URLが漏れると偽リクエストを送れる（予約の偽造など）。URLはLINE Developersコンソール以外に書かない。
被害は部内データに限定され、操作ログ（logs）で追跡可能。

## 4. 機能要件（v1・実装済み）

### 4.1 照会コマンド（応答・無料無制限）
| コマンド | 応答 |
|---|---|
| `今日` | 今日の部室予約一覧＋音出し禁止時間 |
| `今週` | 今週7日分の部室予約一覧 |
| `音出し` | 今日・明日の音出し禁止時間（=それ以外は音出し可能） |
| `ヘルプ` | 使い方 |

### 4.2 LINEから予約作成
- 書式: `予約 6/15 19:00-21:00 バンド名 記入者名`（部室確定枠）
- Webと同じ検証ルールを適用: 15分単位 / 1週間先まで / 同一記入者の未終了確定枠は1件まで / 過去不可
- 成功時は events に追加（Webカレンダーに即時反映）＋操作ログ記録
- 取り消しはWebから（記入者名一致の本人確認があるため）

### 4.3 予約前日リマインド（push・無料枠消費）
- 毎日20時、**翌日に予約がある場合のみ**「通知オン」したトークへ配信
- `通知オン` / `通知オフ` で配信先を登録・解除（Firestore `lineNotifyTargets`）
- 消費目安: 1トークあたり月10〜30通（予約頻度に依存）→ 200通枠内で数グループ運用可能

### 4.4 日程調整（調エモン流用）
- `調整 バンド名` → CHO-EMONテンプレートを複製し、共有リンク（リンクを知っている人は編集可）を返信。
  トークIDとシートIDの対応を Firestore `schedulePolls` に保存
- メンバーは従来通りスプレッドシートに名前と空き時間（○/△）を入力
- `調整結果` → 集計シートを読み、**全員が参加できる時間帯**（集計セルが空欄=不参加者なし）を日付ごとに返信
- 前提: CHO-EMONテンプレートのシートIDを Script Properties `CHOEMON_TEMPLATE_ID` に設定

## 5. 非機能・運用

- データソースは Web アプリと同一の Firestore（リアルタイム一貫性）
- Firestore アクセスはサービスアカウント（JWT/RS256 → REST）。鍵は Script Properties のみに保存（コード・リポジトリに置かない）
- コマンド以外の雑談には反応しない（グループの邪魔をしない）
- 障害時: GASエディタの「実行数」でログ確認
